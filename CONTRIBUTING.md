# Contributing to Textile QMS

Thank you for your interest in contributing to the Textile Quality Management System! We welcome contributions from the community to help make this tool even better.

## How to Contribute

### 1. Reporting Bugs
- Use the GitHub Issue Tracker to report bugs.
- Include detailed steps to reproduce the issue.
- Mention your environment (PHP version, Browser, OS).

### 2. Suggesting Enhancements
- Open an issue to describe the enhancement.
- Explain why it would be useful for the textile industry.

### 3. Pull Requests
- Fork the repository.
- Create a new branch for your feature or fix.
- Ensure your code follows the project's coding style (Clean PHP, standard SQL).
- Test your changes thoroughly.
- Submit a pull request with a clear description of what you've changed.

## Coding Standards
- Use meaningful variable and function names.
- Comment complex logic.
- Ensure SQL queries are optimized and secure (use prepared statements).
- Keep the UI consistent with the existing `style.css` design system.

## Development Workflow
1. Set up a local XAMPP/LAMP environment.
2. Follow the installation steps in the [README.md](README.md).
3. Make your changes.
4. Verify the database schema if you make data structure changes.
